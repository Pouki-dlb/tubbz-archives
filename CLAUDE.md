# CLAUDE.md — The TUBBZ Archive

> Single source of project context, auto-loaded by Claude Code. Written in English.
> Not served to visitors; has no effect on the site. Keep it concise and up to date.

The project has two parts:
- **The public catalog site** (repo root) — archives TUBBZ figurines (vinyl cosplay ducks by
  Numskull) so nothing is lost when a figure sells out, and lets a visitor track their collection
  (owned marks, wishlist, notes). **No account** — everything visitor-side lives in `localStorage`.
- **A local admin tool** (`admin/`) — never deployed; used to edit `data.js` and produce
  correctly-named images. See "Admin tool" below.

## Hard constraints (non-negotiable)

- **100% static**: no framework, no backend, no database, no build step. Must stay free to host
  (GitHub Pages, Netlify, Cloudflare Pages).
- **Must work from `file://`**: never `fetch` a local file (blocked under `file://`). The catalog
  is loaded via `<script src="data.js">`.
- **Figurine `id`s are stable** — never rename one. They key the visitor's collection *and* the
  image filenames; renaming shifts every existing collection.
- **Always escape injected HTML** (`Tubbz.esc` on the site, `esc` in the admin tool).
- **One orange only across the site: `#ee8804`** (`var(--accent)`). Don't introduce another
  orange/amber.
- Site **UI text is English** (character/franchise names kept as-is). The admin tool's UI is French.

## The catalog — `data.js`

Plain JSON wrapped in `window.TUBBZ_DATA = { … };`, loaded via `<script>`. **Never** stored in
`localStorage`. Regenerated by the admin tool: figurines **sorted by franchise then name**, empty
optional fields omitted.

```js
window.TUBBZ_DATA = {
  "meta": {
    "sizes": ["classic", "mini", "xl"],
    "packaging": ["first-edition", "boxed"],
    "labels": {
      "sizes": { "classic": "Classic", "mini": "Mini", "xl": "XL" },
      "packaging": { "first-edition": "First Edition", "boxed": "Boxed" }
    }
  },
  "figurines": [
    {
      "id": "fallout-vault-boy", "name": "Vault Boy", "franchise": "Fallout",
      "number": "42", "releaseYear": "2020", "description": "Vault-Tec mascot.",
      "variants": [
        { "size": "classic", "packaging": "first-edition", "limitedTo": 5000 },
        { "size": "classic", "packaging": "boxed" },
        { "size": "mini",    "packaging": "boxed" }
      ]
    }
  ]
};
```

**Figurine fields**: `id` (required, **unique & stable**), `name` (required), `franchise`
(required), `number?` (shown `#42`, else `—`), `releaseYear?` (else `Unknown`), `description?`
(`\n` preserved, shown italic), `variants` (required).
**Variant fields**: `size` (`classic|mini|xl`), `packaging` (`first-edition|boxed`), `limitedTo?`
(number, shown "Limited to 3,000 units"). List only variants that **actually exist**.
Internal variant key: `"<size>|<packaging>"`. **Max 4 variants per figurine.**

## Image naming (derived from `id` — no paths in `data.js`)

Put `.webp` files in `images/`; the app computes paths from the `id`. Missing file → `placeholder.svg`
via `onerror` (no visible error). Size initials `classic→c`, `mini→m`, `xl→x`; packaging
`first-edition→f`, `boxed→b`.

- **Per size** (bare figurine): `images/<id>-<c|m|x>.webp` — used for the `duck.html` hero and the
  default card image. Helpers `Tubbz.sizeImageFor` (site) / `imageForSize` (admin). **Primary size**
  = `classic` if present, else the single size = the default card image.
- **Per variant** (figurine in its packaging): `images/<id>-<c|m|x><f|b>.webp` (e.g. `-cf`, `-cb`,
  `-mf`, `-xb`) — used for the "Available versions" tiles and the chip-hover preview. Helper
  `variantImageFor`.
- Every image the admin tool produces is **400×400 px, WebP, quality 0.75**.

## Visitor collection (`localStorage`)

Stored under key `tubbz-collection`, browser-only:

```json
{
  "version": 1,
  "owned":    { "fallout-vault-boy": { "classic|first-edition": true } },
  "wishlist": { "fallout-vault-boy": true },
  "notes":    { "fallout-vault-boy": "paid 15€" }
}
```

**Export** downloads exactly this object as `.json`; **Import** restores it (replace, after
confirmation). This is the only backup/transfer mechanism (no account, no server).

## Admin tool (`admin/`, local only — never deployed)

- **Purpose**: edit `data.js` (add/edit/delete ducks) and produce normalized images. You run it
  against a **local copy** of the site, then push `data.js` + `images/` yourself.
- **100% autonomous**: depends only on `data.js`, `images/` (root) and `admin/vendor/` (Cropper.js).
  Never make it depend on `common.js`, `styles.css`, `index.*`, `duck.*`. Helpers stay inline in
  `admin/index.js`; styles in `admin/index.css`.
- **Git-ignored** (`.gitignore`: `admin/`) → install = drop the folder in, uninstall = delete it.
- `admin/index.html` carries **`<base href="../">`** so display URLs (`images/…`, `data.js`) resolve
  from the project root. Disk **writes** use the File System Access API (paths relative to the
  folder you pick), so `<base>` doesn't affect them.
- **Writing**: File System Access API (`showDirectoryPicker`, Chrome/Edge); if unavailable, falls
  back to **download** mode (`data.js` + `.webp` offered as downloads). Keep this fallback working.
- **`id`s**: locked when editing (unlock is an advanced action); on create, auto-generated from
  `franchise + name` via `slugify` and editable until first save.
- Editor renders one image slot per **distinct size** (`#size-images-list`) and flags missing images.

## Conventions & gotchas

- **Variant chip** = ownership indicator (colored + ✓ if owned, greyed otherwise). Hover a card
  chip → the card image becomes that variant's; click → opens the detail page. Max 4 chips.
- **Index card is a `<div>`, not one big link**: image, name, and chips lead to the detail page; the
  franchise name filters that franchise (`?franchise=<name>`, mirrored on `duck.html`).
- **`duck.html` hero** = per-size image + a flip button that cycles sizes when a duck has more than
  one; single-size ducks get no button.
- **Logo gotcha**: the title and tagline lines are justified to end at the same right edge. The
  tagline's `font-size` and `letter-spacing` are tuned so it reaches the title's width naturally —
  **if the title or tagline text changes, recompute those two values** (otherwise justification
  opens big gaps), and keep the header height (~69px).
- New shared logic → `common.js`; page-specific → `index.js` / `duck.js`; admin-specific →
  `admin/index.js`.
